Визуализация вылета #

Игра падает на чтении поля +0x30 у нулевого объекта (this == 0). Нулевой объект появляется потому, что фабрика растрового кэша возвращает NULL при отказе аллокатора, а вызвавший её код передаёт этот NULL в функцию заполнения растра без проверки. Схема показывает путь данных от аллокатора до точки сбоя.

1. Аллокатор SoD 617492h
Выделить пиксельный буфер спрайта SpelScBk.pcx
отказ
возвращает NULL (nothrow)
2. Фабрика sub_10079970
видит: поле [obj+0x30] == 0
возврат NULL
xor eax, eax
3. Инициализатор sub_10103400
mov ecx, eax (без test/jz)
проверки НЕТ
вызов sub_1007A370
4. Точка сбоя 0x1007A370
mov edx, [ecx+30h]
ECX = 0
чтение 0x00000030

Дескриптор растра, который функция sub_1007A370 пытается читать через ECX:

this == ECX
0x00000000
[this+0x24] ширина
не достигнуто
[this+0x28] высота
не достигнуто
[this+0x2C] шаг
не достигнуто
[this+0x30] буфер пикселей
== 0x00000030
адрес сбоя
отказ/сбой = отказ аллокатора, нулевой возврат, точка падения
возврат NULL = корректное действие фабрики, транслирующей отказ
не достигнуто = поля, до которых исполнение не дошло (сбой на первом чтении)

В той же функции инициализатора проверка возврата другой аллокации (operator new) присутствует — значит отсутствие проверки после фабрики именно пропуск, а не сознательное решение.
